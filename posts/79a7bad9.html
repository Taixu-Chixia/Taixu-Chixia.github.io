我的第二篇博客：关于博客的部署过程中的一些问题汇总

2021-01-27 hexo►teach►wrong bug, hexo, teach 0 评论

我将我的博客部署在了gitee上。能看到这里，不出意外都是用的gitee连接
以下问题都发生在我的Windows操作系统下 其他系统出了问题的话看个思路就行，解决方法大同小异

部署问题1
在gitee上新建一个仓库(路径名和用户名相同就不用重复了)
新建后，必须完成仓库的创建过程再进行pull操作(hexo d)

部署问题2
修改_config.yml 文件失败
这里一定要小心，复制粘贴后要注意空格，缩进等问题。最好自己手动敲一遍。

部署问题3
*** Please tell me who you are
你在命令行中执行
git config –global user.email “你的邮箱”
git config –global user.name “你的名字”
能解决这个问题
这个问题可以看到如果不执行这两行
会报错如下
Run

git config –global user.email “you@example.com“
git config –global user.name “Your Name”

to set your account’s default identity.
Omit –global to set the identity only in this repository.

fatal: unable to auto-detect email address (got ‘xxx.(none)’)

部署问题4
第一次接触这方面的同学
可能需要配置自己计算机的SSH公钥

ssh-keygen -t rsa -C “xxxxx@xxxxx.com“
在命令行输入这样格式的命令引号里是邮箱，引号前面有一个空格

第一次回车确认 第二次确认 第三次输入密码 第四次确认密码

然后在C:\Users\你的用户名.ssh\id_rsa.pub这个文件里找到你的SSH公钥
用记事本或者VSCode打开
将文件内的内容复制到gitee的部署公钥设置里(进入个人设置页面—》SSH 公钥添加)
不是在库里添加哦，是在个人设置里添加。

部署问题5
密码错误问题
remote: Incorrect username or password ( access token )
打开电脑的控制面板–>用户账户–>管理Windows凭据
找到普通凭据中自己的账号信息，选择编辑，填入正确的用户名和密码，最后点击保存即可。
这里的用户名指的是你登录gitee用的用户名和密码

部署问题6
部署后没有正常显示
我的是在库设置里更新一下就好了
应该是在gitee pages服务里
